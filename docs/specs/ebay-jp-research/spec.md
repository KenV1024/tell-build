## 0. 最初に読んでほしい重要な前提（3行まとめ）

1. **eBayの「売れた商品（Sold）」データは、誰でも使えるAPIでは取得できません**。公式APIは審査制の限定公開で、個人開発者はまず承認されません。現実的には eBay公式リサーチツール「Terapeak（テラピーク）」を使い、**人が条件を絞って抽出 → CSVで書き出す**のが安全な入口です。
2. **メルカリ・ヤフーフリマ・楽天ラクマには、商品データを取れる公式APIが存在しません**。自動収集（スクレイピング ← プログラムでWebページのデータをかき集める行為）は各社の利用規約に抵触するおそれがあり、法的・アカウント停止リスクがあります。
3. そのため本ツールは **「全自動」ではなく「半自動（人の確認を挟む設計）」** を初期方針とします。リスクの高い自動収集は、あなたが内容を理解したうえで明示的にONにする「オプション扱い」にします。

> ← これらは「できない」という意味ではなく、「安全にやるための入口はここ」という整理です。詳細と代替案は本文と末尾「要確認事項」に記載します。

## 1. 概要

eBayで日本のセラーから売れた商品（Sold Items）をリサーチし、それを国内フリマ（メルカリ／ヤフーフリマ／楽天ラクマ）で仕入れられるかを照合、eBay輸出転売の**利益候補リスト**を半自動で出力するリサーチ支援ツール。仕入れ〜出荷そのものは自動化せず、**「どの商品を狙うべきか」の意思決定材料づくり**に絞る。

---

## 2. 利用シーン（誰が・いつ・何のために）

- **誰が**: オーナー本人（eBay輸出のリサーチ担当。非エンジニア、Windows 11 / Python / GitHub Actions 環境）。
- **いつ**: 週数回のリサーチ作業時。または定期実行（例: 毎朝）で候補リストを受け取る。
- **何のために**: 「eBayで日本発の商品が実際に売れている」という需要の裏付けがある商品に絞り、国内で安く仕入れられるものだけを効率的に見つけ、赤字リサーチの時間を減らすため。

---

## 3. スコープ

### やること（本ツールの責任範囲）
- eBayのSold（販売済み）商品データを**取り込む**（取得方法は後述の複数案。初期は手動エクスポート取込）。
- 取り込んだ商品を国内フリマで検索するための**検索URL・検索キーワードを自動生成**する。
- （オプション）国内フリマの価格候補を取得し、eBay売価と突き合わせる。
- eBay手数料・送料・為替を加味した**利益額／利益率を自動計算**する。
- 利益が出そうな候補を**CSV／スプレッドシート形式でリスト出力**する。

### やらないこと（スコープ外・後工程での勝手な拡張を禁止）
- 実際の**仕入れ（購入）・出品・出荷の自動化**はしない（人が判断・実行）。
- eBayやフリマへの**自動ログイン・自動購入・自動出品**はしない。
- 商品画像のAI鑑定や真贋判定はしない。
- 在庫・受注・会計管理はしない（リサーチ専用）。
- 規約違反・法令違反となる手段の実装はしない（後述のリスク方針に従う）。
- 各サービスがブロックする回避策（CAPTCHA突破、大量並列アクセス等）は実装しない。

## 4. 法的・技術的制約（最重要・必読）

### 4-1. eBay の Sold（販売済み）データ取得
- **Marketplace Insights API**（売れた商品の販売履歴を返す公式API）は「限定公開（Limited Release）」で、**eBayの審査を通過した提携パートナーのみ**。個人・小規模開発者は本番アクセスをまず取得できない。 [出典1][出典2]
- **Browse API**（誰でも使える購入系API）は**現在出品中（アクティブ）の商品のみ**で、Soldは取得不可。 [出典3]
- 旧 **findCompletedItems**（過去の完了商品を取れたAPI）は**提供終了**済み。 [出典2]
- → **現実的な入口**: eBay公式リサーチツール **Terapeak / Product Research**（Seller Hub内、eBayストア契約者が利用可）。**セラーの所在地=日本**で絞り込み、Soldでソートして販売実績を確認・**書き出し**できる。ただしこれは画面ツールでありAPIではないため、**取込はCSV/手入力ベース**になる。 [出典4]

### 4-2. 国内フリマの公式API
- **メルカリ**: 一般向けの商品データ取得APIは**なし**。「メルカリShops API」は自店舗の商品登録・編集用で、他人の出品を収集する用途ではない。 [出典5]
- **ヤフーフリマ（旧PayPayフリマ）**: フリマ商品用の公式APIは**確認できず（実質なし）**。Yahoo!デベロッパーのAPIは「Yahoo!ショッピング」用でフリマとは別物。 [出典6]
- **楽天ラクマ**: 商品データ取得の公式APIは**なし**（楽天市場のAPIは存在するがラクマとは別サービス）。 [出典7]

### 4-3. スクレイピング（自動収集）のリスク
- **メルカリ**: 規約で「提供インターフェイス以外の手法でのアクセス」「無許可の商業的なデータ利用」を禁止する趣旨の条項があり、スクレイピングは抵触リスク大。ログイン状態での実施は規約同意下となりさらにリスク。大量アクセスは業務妨害に問われる可能性も指摘されている。 [出典5]
- **楽天（ラクマ含む）**: 規約で無許可の自動情報取得を禁止。 [出典7]
- **ヤフー系**: 大手ECは同様に自動取得を制限する運用が一般的。 [出典6]
- **本ツールの方針**: リスクの高い経路は既定で**「検索URLを生成して人が目視確認する半自動」**に倒す。完全自動収集は、リスクを理解したうえでの**明示的オプトイン（既定OFF）**とし、低速・少量・キャッシュ非保存を強制する。最終判断（採用するか）はオーナーの自己責任事項として仕様書に明記する。

## 5. システム構成（初期方針: 半自動）

```
[eBay Sold 取込]                     [国内マッチング]              [利益計算]        [出力]
Terapeak等でSold抽出         →  各商品の検索URL/キーワード生成  →  手数料・送料   →  利益候補
 → CSVエクスポート            →  （既定）人が目視で仕入値入力   →  為替を加味    →  CSV/表
 → 本ツールが取り込み            （任意）低速取得オプション        利益率を算出      で出力
```

- **実行環境**: Windows 11 上の Python。定期実行が必要なら GitHub Actions（← クラウドで決まった時間にプログラムを動かす仕組み）。
- **UI**: まずは CLI（コマンド実行）＋ CSV入出力。将来的に簡易画面は任意（要確認）。
- **データ保存**: ローカルのCSV/SQLite（← 1ファイルで完結する軽量データベース）。個人利用のためクラウドDBは既定で使わない。

---

## 6. データフロー（1件の商品がたどる流れ）

1. **入力**: eBay Sold 1件（タイトル、カテゴリ、売価、通貨、送料、販売日、セラー所在地=日本 など）。
2. **正規化**: タイトルから検索キーワードを抽出・整形（ブランド名／型番など）。
3. **国内検索**: メルカリ／ヤフーフリマ／ラクマ向けの検索URLを生成。
   - 既定モード: URLを出力し、人が開いて相場・在庫を確認 → 仕入予定価格を入力。
   - オプションモード（既定OFF・要オプトイン）: 低速で候補価格を取得。
4. **利益計算**: eBay売価 − eBay手数料 − 決済手数料 − 国際送料 − 仕入値、を為替換算して利益額・利益率を算出。
5. **判定**: しきい値（例: 利益率◯%以上）を満たすものに「候補」フラグ。
6. **出力**: 候補リストをCSV/スプレッドシートへ。

## 7. 機能要件

### F1. eBay Sold 取込
- F1-1: Terapeak/Product Research 等で書き出した **CSVを取り込める**（列マッピング設定可）。
- F1-2: セラー所在地=日本 の商品に絞り込める（取込時フィルタ）。
- F1-3: （将来/条件付き）Marketplace Insights APIの承認が取れた場合はAPI取込に切替可能な設計余地を残す。

### F2. 国内マッチング
- F2-1: 商品タイトルから検索キーワードを生成する（不要語の除去、型番抽出）。
- F2-2: メルカリ／ヤフーフリマ／ラクマの**検索URLを生成**する。
- F2-3: 各商品に対し、人が確認した**仕入予定価格・在庫有無を入力**できる欄を用意。
- F2-4:（オプション・既定OFF）低速・少量での価格候補取得。ON時は規約リスク同意の確認を必須化。

### F3. 利益計算
- F3-1: eBay手数料（カテゴリ別の落札手数料）、決済/入金手数料、国際送料、為替レートを設定値として保持。
- F3-2: 利益額・利益率を算出し、しきい値で候補判定。
- F3-3: 為替レートは手動設定または自動取得（要確認）。

### F4. 出力
- F4-1: 候補リストをCSV出力（商品名／eBay売価／想定仕入値／手数料内訳／利益額／利益率／各フリマ検索URL）。
- F4-2: 利益率降順で並ぶ。
- F4-3:（任意）Googleスプレッドシート出力（要確認）。

## 8. 非機能要件

- **実行頻度**: 手動実行を基本。定期実行するなら1日1回程度（低頻度）。
- **アクセス制御**: オプションの取得機能を使う場合、リクエスト間隔を空け（例: 数秒以上）、少量・キャッシュ非保存を強制。
- **コスト**: 追加の有料API契約は既定でなし。eBayストア契約（Terapeak利用のため既に契約想定）以外の新規固定費を発生させない方針。為替APIを使う場合は無料枠内。
- **保守性**: 手数料率・送料・しきい値・列マッピングは**コードを触らず設定ファイルで変更**できる。
- **セキュリティ**: eBay等の認証情報は環境変数/秘密管理に置き、コードやCSVに平文で残さない。

## 9. 受け入れ条件（Acceptance Criteria）

> qa-tester が1項目ずつ実行して合否判定できる粒度で記述。

- [ ] AC-1: サンプルのeBay Sold CSV（10件、うち日本セラー6件）を取り込むと、**日本セラー6件のみ**が処理対象になる。
- [ ] AC-2: 取り込んだ各商品について、メルカリ・ヤフーフリマ・ラクマの**検索URLが3本生成**され、ブラウザで開くと関連商品の検索結果が表示される。
- [ ] AC-3: 検索キーワード生成が、タイトル `"Nintendo Game Boy Color Console Japan Used"` から不要語（Japan/Used等）を除いた妥当なキーワードを出力する。
- [ ] AC-4: 各商品に想定仕入値を入力したCSVを再取込すると、利益額・利益率が計算され出力される。
- [ ] AC-5: 手数料率・国際送料・為替レート・利益率しきい値を**設定ファイルで変更**すると、再計算結果に反映される。
- [ ] AC-6: 利益率しきい値（例:20%）以上の商品だけに「候補」フラグが立ち、出力CSVが**利益率降順**に並ぶ。
- [ ] AC-7: オプションの自動取得機能は**既定でOFF**で、ONにするには設定＋リスク同意フラグの両方が必要（片方だけでは動かない）。
- [ ] AC-8: 出力CSVをExcel/スプレッドシートで開くと文字化けせず（UTF-8 BOM等）、必須列がすべて揃っている。
- [ ] AC-9: 為替・手数料が未設定のまま実行すると、**エラーで止まらず**警告を出し既定値を使う（またはガイドを表示）。
- [ ] AC-10: サンプル一式（入力CSV・設定ファイル・期待出力）が同梱され、`README`の手順どおりで再現できる。

## 10. 実装計画（フェーズ分け）

- **フェーズ1（MVP・最優先）**: CSV取込 → 日本セラー絞り込み → 検索URL生成 → 手入力仕入値での利益計算 → CSV出力。**規約リスクのある自動取得は含めない**。AC-1〜AC-6, AC-8〜AC-10を満たす。
- **フェーズ2**: 設定ファイル整備、キーワード生成の精度改善、為替自動取得、スプレッドシート出力。
- **フェーズ3（条件付き・要判断）**: オプションの低速取得（オプトイン）、または Marketplace Insights API 承認が取れた場合のAPI取込への切替。AC-7を満たす。GitHub Actionsでの定期実行。

> フェーズ1だけでも「需要の裏付けがある商品の国内相場を効率的に調べる」実務は回せる設計にする。
