# 品質管理チェックリスト（Anthropic基準）

**使い方**: forge本体に変更を加えたとき、およびパイプラインで作ったプロダクトを公開する前に、
このリストを上から実施する。「必須」が1つでも❌ならリリース不可。

最終実施日: 2026-07-07（内部アルファ監査）

---

## A. エージェント定義の品質（forge本体を変更したら実施）

| # | 項目 | 必須 | 状態 |
|---|---|---|---|
| A-1 | 各エージェントのdescriptionが「いつ使うか」を明記している | 必須 | ✅ |
| A-2 | 各エージェントに越権禁止事項が書かれている（qa-testerは直さない等） | 必須 | ✅ |
| A-3 | `tools:` で最小権限が設定されている（不要なBash/Edit権限がない） | 必須 | ✅ 2026-07-07対応 |
| A-4 | model/effort指定が `shared/pipeline_rules.md` の表と一致している | 必須 | ✅ |
| A-5 | 出力ファイルのパス・形式が正典（pipeline_rules.md）と一致している | 必須 | ✅ |
| A-6 | ドキュメント間で矛盾がない（CLAUDE.md・正典・コマンド・エージェント定義） | 必須 | ✅ 2026-07-07対応 |

## B. 安全性（Human-in-the-loop / 破壊的操作の防止）

| # | 項目 | 必須 | 状態 |
|---|---|---|---|
| B-1 | ゲート1なしでcode-builderが起動しない導線になっている | 必須 | ✅ |
| B-2 | ゲート2なしでpush・公開・課金操作が実行されない導線になっている | 必須 | ✅ |
| B-3 | 秘密情報（APIキー等）のハードコード禁止がcode-builder定義にある | 必須 | ✅ |
| B-4 | code-builderの書き込み先が `projects/<project>/` に限定されている | 必須 | ✅ |
| B-5 | 差し戻し無限ループの防止策（3往復上限）が機能する | 必須 | ✅ |
| B-6 | 外部コンテンツ（Webページ・ユーザー提供データ）を「指示」ではなく「データ」として扱う注意書きがある | 推奨 | ✅ 2026-07-07対応 |
