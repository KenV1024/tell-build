# TellBuild — 「作りたい」を伝えるだけの開発パイプライン（パブリックベータ）

TellBuildは、Claude Code上で動く**ソフトウェア開発の自動パイプライン**です。
作りたいものを一言伝えると、6体の専門AIエージェントがリレー形式で
「要件ヒアリング → 実装 → テスト → 公開 → 報告 → 改善」を進めます。

あなたがやることは2つだけ:
1. 作りたいものを説明する（質問に答える）
2. 2つの承認ポイントで「OK」を出す（仕様の承認と、公開の承認）

---

## 前提条件

| 項目 | 要件 |
|---|---|
| Claude Code | インストール済みであること（CLI または VS Code拡張） |
| モデルアクセス | **Opus 4.8へのアクセスを推奨**（要件定義・実装・原因分析で使用）。ない場合は下記フォールバック参照 |
| OS | Windows / macOS / Linux（開発・検証はWindows 11で実施） |
| コスト | Claude利用料は利用者負担。小規模プロジェクト1件で目安数百円〜（Opus使用時） |

**Opusが使えない場合**: `.claude/agents/` 内の各ファイルの `model: claude-opus-4-8` を
`claude-sonnet-5` に書き換えれば全編Sonnetで動きます（要件定義の掘り下げ精度は下がります）。

## クイックスタート（3ステップ）

```bash
# 1. クローンして移動
git clone <このリポジトリのURL>
cd tell-build

# 2. Claude Codeを起動
claude

# 3. 作りたいものを伝える
/tell-build my-tool "毎朝の売上CSVを集計してグラフ画像にするツール"
```

あとはエージェントの質問に答えるだけ。仕様書ができたら承認を求められます（ゲート1）。
承認するとコーディング・テストが自動で進み、公開前にもう一度確認されます（ゲート2）。

## コマンド一覧

| コマンド | 用途 |
|---|---|
| `/tell-build <name> "<説明>"` | 新規プロジェクト開始 / 中断したプロジェクトの再開 |
| `/tell-build-status` | 進行中プロジェクトの現在地を確認 |
| `/tell-build-improve <name> "<フィードバック>"` | 完成後の不具合報告・改善要望を投入 |

## 仕組み（ざっくり）

```
あなたの依頼
   ↓
要件ヒアリング（spec-interviewer）─→ 【承認①: この仕様でいい?】
   ↓
実装（code-builder）
   ↓
テスト（qa-tester）──不合格なら実装へ差し戻し（最大3回）
   ↓
【承認②: 公開していい?】─→ 公開（release-manager）
   ↓
完了報告（progress-reporter）──→ あなた
```

- 成果物はすべてファイルに残ります: 仕様書は `docs/specs/`、コードは `projects/`、
  テスト結果・進行状況は `state/`、あなた向け報告は `reports/`
- あなたの承認なしにgit push・Web公開・課金操作が実行されることはありません

## うまくいかないとき（FAQ）

**Q. 途中で止まった/セッションが切れた**
A. `/tell-build <プロジェクト名>` をもう一度実行してください。`state/<プロジェクト名>/pipeline_status.json` に記録された続きから再開します。

**Q. 今どこまで進んでいるかわからない**
A. `/tell-build-status` で全プロジェクトの現在フェーズが見られます。

**Q. テストの差し戻しが3回を超えた**
A. 自動で原因分析担当（improvement-analyst）が起動し、「実装のバグ」か「そもそも仕様に無理があるか」を切り分けて提案します。

**Q. できあがったものが思っていたのと違う**
A. `/tell-build-improve <プロジェクト名> "こう直したい"` と伝えてください。修正か仕様変更かを判定して適切な工程に差し戻します。

「これがライセンス文書だ」と認識する決まりごとになっています。

TellBuildは **MITライセンス** を採用しています。要約すると:
- 誰でも自由に使う・改変する・配布する・商用利用してよい
